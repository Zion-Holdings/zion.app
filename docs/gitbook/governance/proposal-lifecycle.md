# Proposal Lifecycle

1. Draft: authors prepare spec and rationale; optional temperature check
2. Review: formalization, risk analysis, and delegate feedback
3. Active Voting: onchain vote window opens
4. Queued: successful proposals enter timelock
5. Executed: transactions executed by executor
6. Finalized: monitoring and post-mortem recorded

## Metadata
- Title, authors, rationale, parameters touched
- Links: forum/thread, code diffs, audits, simulations
- Versioning: semantic versions and proposal IDs
